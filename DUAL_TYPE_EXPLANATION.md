# 双属性宝可梦的属性关系说明

## 问题：为什么妙蛙种子的攻击优势中显示"草系 2×"？

### 简短回答
**不是草系克制草系**，而是妙蛙种子的**毒系**克制草系！

### 详细解释

妙蛙种子是**草系+毒系**双属性宝可梦，它的攻击优势来自两种属性：

#### 1. 草系技能的克制效果
草系技能克制以下属性（2倍伤害）：
- 水系
- 地面系
- 岩石系

#### 2. 毒系技能的克制效果
毒系技能克制以下属性（2倍伤害）：
- **草系** ← 这就是为什么显示"草系 2×"
- 妖精系

### 属性相克表验证

查看代码中的 `typeMatchups` 对象：

```javascript
const typeMatchups = {
    grass: { 
        water: 2,      // 草系克制水系
        fire: 0.5,     // 草系被火系克制
        grass: 0.5,    // 草系攻击草系效果不佳（0.5倍）
        ground: 2,     // 草系克制地面系
        rock: 2,       // 草系克制岩石系
        // ...
    },
    poison: { 
        grass: 2,      // 毒系克制草系 ← 关键！
        poison: 0.5,   // 毒系攻击毒系效果不佳
        // ...
    }
};
```

### 实际战斗中的应用

当妙蛙种子对战草系宝可梦时：

1. **使用草系技能**（如藤鞭）：
   - 效果不佳（0.5倍伤害）
   - 因为草系攻击草系是 0.5×

2. **使用毒系技能**（如毒粉）：
   - 效果拔群（2倍伤害）
   - 因为毒系克制草系

### 为什么会产生混淆？

在"攻击优势"区域中，我们显示的是**该宝可梦能够克制的所有属性**，包括：
- 草系技能能克制的属性
- 毒系技能能克制的属性

所以当看到"草系 2×"时，实际上是指：
> "妙蛙种子可以使用毒系技能对草系宝可梦造成2倍伤害"

而**不是**：
> "妙蛙种子的草系技能克制草系"（这是错误的）

### 改进措施

为了减少混淆，我们在战术建议中添加了说明：

```
📌 妙蛙种子 拥有 草系、毒系 双属性，攻击优势来自两种属性的技能
```

这样用户就能理解：
- 攻击优势列表中的属性是**所有技能类型**的综合结果
- 需要根据技能类型选择合适的招式

### 其他双属性宝可梦的例子

假设有一个**水系+飞行系**的宝可梦：

**攻击优势可能包括**：
- 火系（水系克制）
- 地面系（水系克制）
- 岩石系（水系克制）
- 草系（飞行系克制）
- 虫系（飞行系克制）
- 格斗系（飞行系克制）

**防御劣势可能包括**：
- 电系（4倍！因为水系和飞行系都被电系克制）
- 岩石系（4倍！因为飞行系被岩石系克制2倍，水系对岩石系没有抵抗）

### 总结

| 问题 | 答案 |
|------|------|
| 草系克制草系吗？ | ❌ 不克制，效果不佳（0.5×） |
| 毒系克制草系吗？ | ✅ 克制，效果拔群（2×） |
| 妙蛙种子能克制草系吗？ | ✅ 能，使用毒系技能 |
| 妙蛙种子的草系技能克制草系吗？ | ❌ 不克制，效果不佳 |

### 建议

在实际对战中：
1. 查看"攻击优势"了解可以克制哪些属性
2. 根据对手的属性选择合适的**技能类型**
3. 例如：妙蛙种子对战草系宝可梦时，优先使用毒系技能（毒粉）而不是草系技能（藤鞭）

### 未来改进方向

可以考虑在UI中添加更详细的信息：
- 标注每个克制关系来自哪个属性
- 例如："草系 2× (来自毒系技能)"
- 或者分别显示每种属性的克制关系
